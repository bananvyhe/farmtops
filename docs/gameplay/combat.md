# Combat

## Боевая цель шарда

Полный цикл шарда должен укладываться в 2-5 часов целевой сессии и состоять из:

- набора XP на мобах;
- добычи лечащих предметов и ресурсов;
- конверсии ресурсов в энергию и усиления;
- подготовки к бою с боссом;
- синхронизации персонажей при совпадении праймов;
- кооперативного убийства босса.

Это не action-heavy управление. Пользователь в основном наблюдает, как персонажи качаются, фармят и при совпадении праймов начинают двигаться к общей цели.

## Основные сущности боя

- игрок;
- бот игрока;
- обычные мобы;
- элитные мобы;
- мировой босс;
- healing pickups;
- расход энергии;
- урон, защита, скорость атаки, sustain.

## Правила scaling

С увеличением числа игроков должны расти:

- суммарный HP пул мобов;
- численность паков;
- плотность волн около центра;
- HP, armor, phase count и enrage босса;
- требование по суммарному DPS и живучести группы.

С увеличением числа игроков не должно расти линейно все подряд. Нужен мягкий scaling, чтобы:

- 2 игрока могли закрывать шард медленно, но надежно;
- 3-4 игрока были оптимумом;
- 5 игроков получали ускорение без развала баланса;
- соло-игрок не закрывал босса обычным фармом.

## Формула баланса первой версии

Практический старт:

- world tier определяется followers bucket и размером matched group;
- mob HP = base * map_tier * group_scalar;
- boss HP = base * map_tier * group_scalar * coop_multiplier;
- resource density = base * sqrt(group_size);
- recovery item density = base * ln(group_size + 1).

`coop_multiplier` должен быть выше линейного, чтобы босс требовал нескольких игроков.

## XP и уровни

Ориентир: tempo прогрессии как в Lineage 2, то есть:

- ранние уровни берутся быстро;
- затем стоимость уровней заметно растет;
- midgame дает ощущение долгого фарма;
- финальная часть сессии уходит на подготовку к боссу, а не только на бег по карте.

Смысл уровня не символический:

- уровень показывает реальную глубину фарма и длительность участия в симуляции;
- уровень служит индикатором того, насколько персонаж готов к совместной цели;
- уровень показывает, сколько прогресса накоплено до момента, когда совпадение по прайму даст группе двинуться вперед.

Для первой версии подходит экспоненциально-ступенчатая кривая:

- 1-10 уровни: быстрый onboarding;
- 11-25 уровни: основная фарм-зона;
- 26+ уровни: дорогой прирост для статов и доступа к boss attempt.

Точный numeric table нужно вынести в отдельный balancing doc после ввода боевого прототипа.

## Bot runtime

Игра должна уметь играть сама в себя.

Базовый bot behavior:

- двигаться к ближайшей выгодной цели;
- добивать безопасных мобов;
- подбирать healing items при риске смерти;
- избегать босса до минимального combat threshold;
- присоединяться к бою с боссом при наличии союзников в центре;
- при overlap праймов смещаться от локального фарма к общей цели.

Bot runtime в первой версии завязан на prime simulation:

- пользователь отмечает часы в профиле;
- в эти часы система активирует его бот-участие;
- личное присутствие пользователя желательно, но не обязательно;
- если нашлись другие участники с тем же интересом и пересекающимся праймом, боты начинают показывать групповое продвижение.

## Условия успешного шарда

- минимум несколько участников реально появились в окне близкого времени;
- или были представлены активными ботами в окне близкого времени;
- шард накопил нужный уровень подготовки;
- совпадение праймов перевело ботов к общей цели;
- босс убит совместными усилиями.

Если игроки не совпали по прайму, шард не считается полноценной успешной сессией, даже если боты удерживали экономику.
