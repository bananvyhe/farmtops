# Worldgen Overview

## Цель

Каждый шард должен ощущаться как отдельный мир, вдохновленный конкретной игрой из каталога, но при этом оставаться процедурно генерируемым и детерминированным через seed.

## Источники входных данных

- `game.name`
- число followers
- список участников шарда
- overlap по prime grid
- shard difficulty tier
- текущий шаблон промта для AI-инфраструктуры проекта

## Что генерируется

- имя шарда;
- biome и визуальная тема;
- карта;
- стартовые позиции игроков на удаленных концах карты;
- центральная арена босса;
- кольца мобов вокруг босса;
- resource nodes;
- точки recovery item;
- названия уникальных ресурсов;
- пиксельные ассеты: tiles, props, mobs, boss, pickups, icons.

## Пространственная модель

Базовый layout:

- игроки спавнятся по периферии;
- центр карты зарезервирован под босса;
- между краями и центром размещаются пояса мобов и ресурсных зон;
- безопасные pockets редки и кратковременны;
- маршруты к центру должны давать выбор между фармом XP и риском ускоренного выхода к боссу.

## Scaling по числу игроков

Количество участников влияет на:

- размер карты;
- плотность мобов;
- число resource nodes;
- запас HP и фазы босса;
- required cooperative DPS;
- объем healing pickups.

Принцип:

- 1 игрок может фармить外围 и готовить прогресс, но не должен стабильно закрывать босса.
- 2-5 игроков являются целевой группой для полного цикла фарма и убийства босса.
- 6+ игроков допускаются как overflow режим, но баланс первой версии таргетируется на 2-5.

## AI pipeline

### Тексты и названия

Через существующую инфраструктуру сайта генерируются:

- название шарда;
- название уникального shard resource;
- краткие flavor-описания biome, мобов и босса;
- наименования редких предметов.

Для этого переиспользуется pipeline, который уже работает с определением игр, но меняется prompt и схема результата.

### Pixel-art assets

Через tunnel к локальной машине генерируются:

- tileset;
- спрайты мобов;
- спрайт босса;
- resource nodes;
- consumables;
- icons.

Рекомендуемый pipeline:

- Stable Diffusion 1.5
- LoRA под pixel art
- postprocess: downscale
- nearest neighbor upscale or display
- palette reduction до 8-16 цветов

## Кэширование

AI-результаты должны кэшироваться на двух уровнях:

- по игре и biome template;
- по конкретному shard seed.

Это снижает стоимость и позволяет при повторных заходах быстро восстанавливать мир.
