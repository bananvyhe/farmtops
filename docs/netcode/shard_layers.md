# Shard Layers

## Текущая модель

Шард теперь общий для всей игры: `1 game = 1 shared shard = 1 active layer`.

- все игроки игры входят в один и тот же шард;
- все участники находятся в общем слое;
- map/resources/mobs/boss state едины для всех;
- ручного выбора слоя на фронте больше нет.

## Почему так

Цель - онлайн-ощущение общего мира. Пока существовало разбиение на слои, игроки видели разные симуляции и не могли полноценно взаимодействовать в одном runtime.

## Правила входа

1. При `POST /api/games/:game_id/shard` сервер ищет существующий шард игры.
2. Если шард найден, пользователь добавляется в его `default_layer`.
3. Если шард отсутствует, он создается и пользователь сразу попадает в `default_layer`.

## Ограничения и масштабирование

- Текущий capacity общего слоя увеличен (`200`) как компромисс для MVP.
- При дальнейшем росте онлайна нужно переходить к zone/instance архитектуре, но с сохранением общего world-state для ядра мира.
